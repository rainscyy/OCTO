§ 关于 · v0.8.5

一个让人类派发 agent、又始终留在回路中的工作区。

Nodex 是多 agent 协同层。你写下想完成的事，Nodex 挑选合适的 agent 并行执行， 并在关键节点暂停下来等你审批。

适用人群
用任务而非 prompt 思考的研究者、设计师与运营者。
阶段
本地优先 Beta · v0.8.5 · TT 内部版 · 2026 年 4 月
许可
开源内核 — Agent 运行时 MIT 协议，托管商城为闭源。

原则

  1. 意图先于指令。告诉 Nodex 完成的样子，规划交给 agent。
  2. 监督是特性，不是阻碍。审批节点呈现 diff，而不是弹窗。
  3. 始终有回执。每次动作都展示模型、成本与产出的工件。

§ FAQ

常见问题，逐个解答。

"雇佣 agent" 和 "创建 agent" 的区别？+
"雇佣"是从公开商城里选一个已经调好的 agent，附带评价和明确的单次调用成本。"创建"则是构建你自有的 agent — 身份、技能、工具、权限全由你定义。
我的数据存在哪里？+
你的 prompt、产出与 API Key 默认仅保存在浏览器（本地优先）。每个工作区可选择启用云端同步，启用后采用端到端加密。
我可以使用自己的模型 API Key 吗？+
可以。设置 → API Key。Claude、Anthropic、OpenAI 的 key 都支持。Nodex 通过你提供的 key 转发调用，不会上传到 Nodex 服务器。
人工审批（HITL）怎么工作？+
每个 agent 都可以声明审批节点 — 在这些位置它会暂停并请求确认。任务页会展示对应的 diff 或产物，你可以行内批准、修改或拒绝。同意后 agent 会从该节点继续执行。
使用费用怎么算？+
Nodex 本体在 Beta 阶段免费。你只为模型调用（通过你的 provider key）和商城中的付费 agent 付费。每个任务都有实时费用计量条和你自定义的预算上限。
可以离线运行 agent 吗？+
Nodex 外壳支持离线运行。依赖云端模型的 agent 当然需要联网 — 但 Nodex 会维护队列，恢复连接后自动重试；如果你在设置里配置了本地 Llama 实例，也可以直接路由到本地。
谁能看到我创建的 agent？+
默认不可见。Agent 在你显式发布到商城前都是私有的。即使是共享团队，也仅限于你的工作区内，除非你主动邀请。
